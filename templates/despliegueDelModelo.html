Despliegue del Modelo

En esta sección se describe cómo se llevó el modelo de clasificación de flores desde el entorno local de desarrollo hasta un entorno de producción accesible mediante una aplicación web construida con Flask y desplegada en la plataforma Render.

Implementación en Producción

Esta aplicación se encargó de:

- desplegar un sistema de login

- exploner en un menu las faces de la metodologia CRISP-ML

- exponer el modelo SVM entrenado para clasificar imágenes de flores .

- Cargar los archivos del modelo (modelo_flores_svm.pkl, scaler.pkl, encoder.pkl).

- Exponer una interfaz donde el usuario puede subir una imagen.

- Preprocesar la imagen (redimensionar, convertir a escala de grises, normalizar).

- Ejecutar la predicción con el modelo.

- Mostrar la clase de flor resultante y el nivel de confianza.

Despliegue en Render

Para desplegar la aplicación en Render, se siguieron estos pasos:

- El proyecto se subió a un repositorio de GitHub.

- en el proyecto se creó un archivo requirements.txt con todas las dependencias necesarias.

- Se configuró un servicio web en Render usando el plan gratuito.

- Render clonó el repositorio, instaló dependencias y levantó el servidor Flask.

- La aplicación quedó disponible mediante una URL pública.
